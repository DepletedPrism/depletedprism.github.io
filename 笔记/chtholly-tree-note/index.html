珂朵莉树学习笔记

2019-06-01
  • 笔记
  • Data Structure
 - Total Views

I initially named it as “Old Driver Tree” ( Which is my codeforces ID ).
(But now I call it Chtholly Tree~).

—— ODT in Codeforces Round #449 Editorial

起源

珂朵莉树（Chtholly Tree），老司机树又称（Old Driver Tree），因此我选择叫它珂朵莉树。起源于 CF896C Willem, Chtholly and Seniorious。

综述

用 std::set 将值相同的区间合并为一个节点。
然而在官方题解里，是用 map 实现的，不过主流还是使用 set。
一看就相当地暴力，其实真的很暴力 = =
所需前置知识为 STL 的使用。
只有在数据随机生成时，珂朵莉树的复杂度才是正确的，因此很容易被卡。对于一些数据比较水的题目，可能拿到 AC，甚至吊打标程。但是这不能改变其复杂度错误的事实，所以还是去走人间正道吧。
不过可以拿珂学去骗分啊，对于有区间赋值要求的题目，可以用珂朵莉树搞一搞，没准就过了（雾

  • split
    split 将包含 pos 的 [L, R] 分裂为 [L, pos-1]（或者[L, pos)） 和 [pos, R]，并返回指向后者的迭代器

以后就可以把 [L, R] 上的操作，转化为 [split(L), split(R+1)) 上的操作了

  • assign
    assign 用于区间赋值。这也是珂朵莉树能够保持复杂度的关键。

复杂度

以原题（CF896C Willem, Chtholly and Seniorious）为例。
每个操作有四分之一的概率出现，即有四分之一的操作为 assign，因此 set 的规模会不断下降，接近 $O(m \log n)$。
我知道这个解释看起来很水，不过官方题解里有更为详尽的证明
